## Why

Сейчас система не имеет публичного механизма для регистрации новых пользователей. Пользователи создаются только через seed скрипты в процессе инициализации базы данных. Это препятствует работе приложения в production, где требуется возможность самостоятельной регистрации пользователей через пользовательский интерфейс.

## What Changes

- **Новый endpoint:** `POST /api/v1/register` для самостоятельной регистрации пользователей
- **Валидация входных данных:** Проверка формата и ограничений для email, username и пароля (минимальная длина, сложность)
- **Проверка уникальности:** Валидация уникальности email и username перед сохранением в БД
- **Хеширование пароля:** Использование bcrypt (cost factor = 12) для безопасного хранения паролей
- **Rate limiting:** Защита endpoint от abuse с ограничением количества попыток регистрации по IP адресу
- **Audit logging:** Логирование всех попыток регистрации (успешных и ошибок) в таблицу audit_logs
- **Ошибки валидации:** Информативные ответы об ошибках при некорректных данных (duplicate email/username, слабый пароль и т.д.)

## Capabilities

### New Capabilities

- `user-registration`: Публичный endpoint для самостоятельной регистрации новых пользователей с полной валидацией и безопасностью

### Modified Capabilities

- `api`: Расширение API спецификации с добавлением нового публичного endpoint POST /api/v1/register
- `security`: Добавление требований безопасности для публичного endpoint (rate limiting, валидация паролей, защита от enumeration)

## Impact

**Новые файлы:**
- `app/api/v1/register.py` - реализация endpoint регистрации

**Изменяемые файлы:**
- `app/main.py` - регистрация нового роутера в FastAPI приложении
- `app/schemas/user.py` - дополнительная схема для request регистрации (UserRegister с требованиями к паролю)
- `app/services/user_service.py` - возможно добавление метода для создания пользователя с валидацией

**Возможные изменения:**
- Миграция БД (если требуются новые поля в User модели) - вероятно, не требуется
- Конфигурация rate limiting (параметры в config.py)

**API изменения:**
- Новый публичный endpoint: `POST /api/v1/register`
- Вход без аутентификации (public)
- Возврат созданного пользователя с его ID

**Новые зависимости:**
- Нет (используются существующие passlib, pydantic, sqlalchemy)

**Влияние на безопасность:**
- Требуется rate limiting для защиты от brute-force атак на подбор usernames/emails
- Требуется валидация пароля (минимальная длина, возможно требование на сложность)
- Требуется логирование всех попыток регистрации для аудита
