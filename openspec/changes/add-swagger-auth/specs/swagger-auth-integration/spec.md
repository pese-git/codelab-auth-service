# Спецификация интеграции аутентификации в Swagger UI

## ADDED Requirements

### Requirement: Схема безопасности OAuth2 Bearer в Swagger UI

Приложение FastAPI ДОЛЖНО поддерживать схему аутентификации OAuth2 Bearer в документации Swagger UI, позволяя пользователям авторизовываться используя AccessToken из системы аутентификации.

#### Scenario: Кнопка Authorize отображается в Swagger UI

- **WHEN** пользователь открывает Swagger UI по адресу `/docs`
- **THEN** кнопка "Authorize" ДОЛЖНА быть видна в верхнем правом углу интерфейса

#### Scenario: Поле ввода токена появляется после нажатия Authorize

- **WHEN** пользователь нажимает кнопку "Authorize"
- **THEN** ДОЛЖНО появиться диалоговое окно с полем для ввода Bearer токена

#### Scenario: Bearer токен отправляется в заголовках запроса

- **WHEN** пользователь вводит токен в диалоговое окно авторизации и делает запрос к защищённому эндпоинту
- **THEN** токен ДОЛЖЕН быть отправлен в заголовке `Authorization: Bearer <token>`

### Requirement: Защищённые эндпоинты отмечены в документации

Все защищённые эндпоинты в документации API ДОЛЖНЫ быть визуально отмечены как требующие авторизации.

#### Scenario: Защищённый эндпоинт показывает иконку замка

- **WHEN** пользователь смотрит защищённый эндпоинт в Swagger UI (например, `/api/v1/sessions`)
- **THEN** иконка замка ДОЛЖНА появиться рядом с методом эндпоинта, указывая на необходимость аутентификации

#### Scenario: Описание эндпоинта включает требование безопасности

- **WHEN** пользователь развёртывает защищённый эндпоинт в документации Swagger
- **THEN** секция "Security" ДОЛЖНА указать, что требуется OAuth2 Bearer токен

### Requirement: Интеграция AccessToken с существующей системой аутентификации

Схема OAuth2 Bearer в Swagger ДОЛЖНА работать с существующей JWT-based системой AccessToken без нарушения существующих потоков аутентификации.

#### Scenario: Токен из Swagger валидируется существующей системой

- **WHEN** пользователь получает AccessToken из эндпоинта `/oauth/token` и вводит его в Swagger UI
- **THEN** токен ДОЛЖЕН успешно авторизовать запросы к защищённым эндпоинтам в Swagger

#### Scenario: Существующие зависимости аутентификации остаются без изменений

- **WHEN** проверяются существующие зависимости кода в `app/core/dependencies.py`
- **THEN** никакие существующие зависимости аутентификации НЕ ДОЛЖНЫ быть изменены или нарушены

### Requirement: Документация Swagger обновлена информацией об авторизации

Документация Swagger UI ДОЛЖНА содержать информацию о том, как получить AccessToken.

#### Scenario: Эндпоинты OAuth задокументированы

- **WHEN** пользователь смотрит документацию Swagger
- **THEN** эндпоинт `/oauth/token` ДОЛЖЕН быть виден и задокументирован со всеми требуемыми параметрами и форматом ответа

#### Scenario: Структура токена объяснена

- **WHEN** пользователь смотрит примеры ответов для эндпоинта токена
- **THEN** ответ ДОЛЖЕН содержать `access_token`, `token_type`, `expires_in` и другие обязательные поля с объяснениями
