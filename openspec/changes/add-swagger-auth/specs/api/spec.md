# Дельта-спецификация API - Интеграция Swagger Auth

## MODIFIED Requirements

### Requirement: Документация Swagger UI

Приложение ДОЛЖНО предоставлять интерактивную документацию API через Swagger UI, которая поддерживает OAuth2 Bearer аутентификацию для тестирования защищённых эндпоинтов.

#### Scenario: Swagger UI доступен с поддержкой авторизации

- **WHEN** пользователь открывает `/docs`
- **THEN** Swagger UI ДОЛЖЕН отображаться с кнопкой "Authorize" для ввода AccessToken

#### Scenario: Защищённые эндпоинты помечены в документации

- **WHEN** пользователь просматривает список эндпоинтов в Swagger UI
- **THEN** все защищённые эндпоинты (требующие Bearer token) ДОЛЖНЫ быть визуально отмечены иконкой замка

#### Scenario: Авторизованные запросы отправляют токен

- **WHEN** пользователь авторизуется в Swagger UI и выполняет запрос к защищённому эндпоинту
- **THEN** запрос ДОЛЖЕН содержать заголовок `Authorization: Bearer {token}` автоматически

#### Scenario: Примеры значений токенов в документации

- **WHEN** пользователь просматривает эндпоинт `/api/v1/auth/oauth/token` в Swagger
- **THEN** ответ ДОЛЖЕН включать реальные примеры структуры `access_token`, `token_type`, `expires_in` и `scope`

### Requirement: Базовый URL и версия API в документации

Документация API ДОЛЖНА четко указывать базовый URL и версию API для авторизации.

#### Scenario: Базовый URL отображается в Swagger

- **WHEN** пользователь открывает Swagger UI
- **THEN** базовый URL `http://auth-service:8003/` или `https://auth.codelab.local/` ДОЛЖЕН быть указан и примени́м в Swagger

#### Scenario: Версия API v1 отображается

- **WHEN** пользователь смотрит информацию об API в Swagger
- **THEN** версия API `v1` ДОЛЖНА быть видна в метаинформации документации
