## Why

В настоящее время Swagger UI в нашем FastAPI приложении не предоставляет возможность авторизации с помощью Bearer токена (AccessToken). Это затрудняет тестирование защищенных эндпоинтов через интерактивную документацию API. Разработчикам и тестировщикам приходится использовать внешние инструменты (curl, Postman) для работы с защищенными эндпоинтами, что снижает эффективность разработки и тестирования.

## What Changes

- Добавление схемы безопасности OAuth2 Bearer в конфигурацию FastAPI приложения
- Настройка Swagger UI для отображения кнопки "Authorize" с возможностью ввода AccessToken
- Интеграция существующей системы аутентификации с Swagger UI
- Обновление документации API для указания защищенных эндпоинтов

## Capabilities

### New Capabilities
- `swagger-auth-integration`: Интеграция схемы безопасности OAuth2 Bearer в Swagger UI для авторизации через AccessToken

### Modified Capabilities
- `api`: Обновление конфигурации FastAPI приложения для поддержки схемы безопасности в документации

## Impact

- **Затронутый код**: 
  - `app/main.py` - добавление схемы безопасности в FastAPI приложение
  - `app/core/dependencies.py` - возможно потребуется обновление зависимостей для Swagger
- **API**: Все защищенные эндпоинты будут помечены как требующие авторизации в Swagger UI
- **Зависимости**: Нет новых зависимостей, используются встроенные возможности FastAPI
- **Системы**: Swagger UI получит функциональность авторизации
